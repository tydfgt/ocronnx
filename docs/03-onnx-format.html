3. ONNX 格式详解

ONNX = Open Neural Network Exchange（开放神经网络交换格式）

3.1 为什么需要 ONNX？

深度学习框架很多：PyTorch (Meta)、TensorFlow (Google)、PaddlePaddle (百度)、MXNet 等。每个框架有自己的专有模型格式，互不兼容。

类比电器工程：就像不同品牌的 PLC 有各自的编程语言和文件格式——西门子用 LAD/STL，三菱用梯形图。ONNX 就像是IEC 61131-3 标准——提供一个通用的中间表示。

模型格式对比

框架专有格式互操作
PaddlePaddle.pdmodel / PIR❌ 只能 PaddlePaddle 用
PyTorch.pt / .pth❌ 只能 PyTorch 用
TensorFlow.pb / SavedModel❌ 只能 TensorFlow 用
ONNX.onnx✅ 多引擎通用

3.2 ONNX 的工作原理

ONNX 定义了一套标准算子（operator），如 Conv、Relu、Add、MatMul 等。模型被表示为一个计算图（有向无环图），节点是算子，边是数据流。

PaddlePaddle 模型
  ↓ paddle.onnx.export()
  ↓ 将 Paddle 算子 → 映射到 ONNX 标准算子
ONNX 模型 (.onnx)
  ↓ 可以被以下引擎加载:
  ├── OpenCV DNN (本项目用)  ← 最轻量
  ├── ONNX Runtime            ← 微软维护, 全平台
  ├── TensorRT                ← NVIDIA, 最快但编译慢
  └── OpenVINO                ← Intel, CPU 优化

3.3 ONNX Opset 版本

ONNX 标准持续演进，每个版本叫一个 opset（算子集版本）。

  • opset 11: 基础算子，广泛兼容
  • opset 14: 本项目使用，兼容 OpenCV 4.x
  • opset 17+: 新增 LayerNorm、GridSample 等

版本选择原则：用目标推理引擎支持的最高版本。本项目用 opset 14。

3.4 本项目中的 ONNX 导出流程

# Step 1: 加载 PIR 模型
det_model = paddle.jit.load('model_dir/inference')  # PIR格式

# Step 2: 导出 ONNX
paddle.onnx.export(
    det_model,
    'output.onnx',
    input_spec=[paddle.static.InputSpec(shape=[1,3,-1,-1], dtype='float32')],
    opset_version=14
)

# Step 3: 简化 (使 OpenCV DNN 可加载)
import onnx; from onnxsim import simplify
model = onnx.load('output.onnx')
model_simp, check = simplify(model)
onnx.save(model_simp, 'output_sim.onnx')

3.5 为什么需要 onnxsim 简化？

OpenCV DNN 的 ONNX 解析器并不支持所有 ONNX 算子。原始 ONNX 包含一些 OpenCV 不认识的算子组合（如某些 Reshape 模式）。onnxsim 会：

  • 常量折叠 (Constant Folding)
  • 消除冗余算子
  • 合并连续操作
  • 简化计算图结构

简化后模型大小不变，但计算图更简洁，OpenCV DNN 可以正常加载。

简化前后模型输出数值误差 < 0.0002，精度无损失。

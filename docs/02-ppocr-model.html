2. PP-OCRv5 模型详解

PP-OCRv5 是百度 PaddleOCR 团队的第五代 OCR 模型系列。本文面向电器工程师，用尽量少的数学讲清楚。

2.1 什么是"模型"？

在 AI 里，"模型"就是一个函数：

输出 = 模型(输入)

类比电器工程：模型就像一个专用芯片——给它输入信号，它输出处理结果。芯片内部有大量参数（电阻、电容值），模型内部有大量权重参数（浮点数字）。

PP-OCRv5 由 两个独立模型 组成：

模型类比功能大小
检测模型视觉定位芯片找出图中文字位置4.8MB
识别模型字符解码芯片读出文字内容17MB

2.2 Mobile vs Server 版本

Mobile (本项目用)Server
检测大小4.8MB88MB
识别大小17MB更大
速度快慢
精度中 (够用)高
适用边缘设备/Jetson服务器/GPU集群
Mobile 版专为移动端和嵌入设备设计，在 Jetson 8GB 上是最佳选择。Server 版精度高 ~2-3%，但模型大 10 倍，推理慢很多。

2.3 检测模型内部结构

检测模型用 DBNet (Differentiable Binarization Network)：

输入: 图片 (3通道, H×W)
  ↓
Backbone: MobileNetV3 (轻量 CNN) ← 提取特征
  ↓
Neck: FPN (Feature Pyramid Network) ← 融合多尺度特征
  ↓
Head: 输出概率图 (1通道, H×W) ← 每个像素是文字的概率
  ↓
后处理: 阈值二值化 → 轮廓提取 → 文本框

详细原理见 第4章 DBNet 检测。

2.4 识别模型内部结构

识别模型用 SVTR (Single Visual Transformer for Recognition)：

输入: 裁剪文字区域 (3通道, 48×W)
  ↓
Patch Embedding: 把图片切成小块，每块转成向量
  ↓
Transformer Encoder: 多层自注意力 ← 理解字符间关系
  ↓
CTC Head: 全连接层 → 输出每个位置的字符概率
  ↓
后处理: CTC 解码 → 文字内容

详细原理见 第5章 SVTR 识别。

2.5 模型文件格式: PIR

PaddlePaddle 3.x 使用新的 PIR (Paddle Intermediate Representation) 格式：

  • inference.json — 模型结构描述（约 200KB～400KB）
  • inference.pdiparams — 模型权重参数（检测 4.7MB，识别 16.5MB）

这与旧版 .pdmodel 格式不同。PIR 是 PaddlePaddle 3.x 的统一中间表示。

⚠️ PIR 模型加载方式特殊：paddle.jit.load('dir/inference')（传文件前缀），不能用 paddle.jit.load('dir')。

2.6 字符集 (Dictionary)

识别模型输出 18385 维向量，对应 18385 个字符类别：

索引含义示例
0blank (CTC空白标记)—
1空格' '
2全角空格'\u3000'
3~6832常用汉字 + 生僻字一乙二十丁厂七卜...
6833~18384英文/数字/标点/特殊符号/emojiA-Z, 0-9, 😀😁😂...

共 18385 个字符，覆盖了现代中文 OCR 的几乎所有需求。

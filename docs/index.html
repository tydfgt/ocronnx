1. 为什么做这个项目

背景

在 Jetson Orin Nano 上做中文 OCR，PaddleOCR 是最成熟的开源方案。但 PaddlePaddle 官方至今未发布 JetPack 6 (L4T 36.x) 的 GPU 预编译包。

已尝试的方案

方案问题速度
PaddleOCR Python CPUARM 无 MKL/MKLDNN → 崩溃❌
C++ Paddle Inference (JetPack 5预编译)TRT不兼容, FP16更慢~5.5s
从源码编译 PaddlePaddle4h+, 8GB内存不足❌
ONNX + OpenCV DNN (本项目)—~1.0s ✅

核心思路: 将 PP-OCRv5 导出为 ONNX 通用格式，用 Jetson 上已有的 OpenCV DNN + CUDA 做推理，完全绕过 PaddlePaddle GPU 运行时。

2. 基础知识详解

2.1 现代 OCR 的两阶段流程

输入图片 → [文字检测] → 文字位置 → [文字识别] → 文本内容

文字检测 (Detection): 在图中找出"哪里"有文字，输出坐标框。
文字识别 (Recognition): 把每个框里的图"读"出来，输出文字。

2.2 PP-OCRv5 模型家族

子模型架构大小功能
PP-OCRv5_mobile_detDBNet (CNN骨架)4.8MB找出文字位置
PP-OCRv5_mobile_recSVTR (Transformer骨架)17MB识别文字内容

Mobile 版专为边缘设备设计，体积小速度快。

2.3 ONNX — 通用模型格式

不同框架各有专有格式。ONNX (Open Neural Network Exchange) 提供中间表示，让模型在框架间迁移：

PaddlePaddle → paddle.onnx.export() → ONNX → OpenCV DNN / ONNX Runtime / TensorRT

2.4 PIR 格式 (PaddlePaddle 3.x 新格式)

PP-OCRv5 模型文件:

  • inference.json — 模型结构 (序列化的 PIR program)
  • inference.pdiparams — 模型权重

这与旧版 .pdmodel 格式不同。

⚠️ PIR 不能用 paddle.jit.load(model_dir) 加载（报 KeyError: 'forward'）。必须传文件前缀: paddle.jit.load(f'{dir}/inference')

2.5 为什么用 OpenCV DNN

Jetson 已自编译 OpenCV 4.10.0 + CUDA（含 DNN CUDA 后端）:

  • ✅ 无需额外推理引擎
  • ✅ 直接利用 CUDA 加速
  • ✅ 无需 PaddlePaddle 运行时

局限: ONNX 算子支持不如 ONNX Runtime 全面，需 onnxsim 简化模型。

2.6 DBNet 文字检测原理

  1. 图片缩放到 960px (长边) → 归一化
  2. CNN 输出概率图 (每个像素是文字的概率)
  3. 阈值 0.3 二值化 → 白色=文字
  4. OpenCV findContours → 文本框
  5. Vatti 裁剪算法 (unclip) 扩展文本框
DBNet 要求输入尺寸是 32 的倍数（下采样 1/32），否则内部 Add 操作维度不匹配。

2.7 SVTR 文字识别原理

  1. 裁剪文字区域 → 缩放高度到 48px → 归一化至 [-1, 1]
  2. Transformer 编码器提取特征 → 输出每个位置的字符概率
  3. CTC 解码处理不定长序列
识别模型 max_imgW 默认 3200（不是 320）。设为 320 会导致长文本无法识别。

2.8 CTC 解码详解

模型输出: (T, 18385) — T 个时间步 × 18385 个字符概率。

解码规则: ① 去重 ② 去 blank (索引0)

例: [b, 登, 登, b, 机, b, 牌] → "登机牌" (b=blank)

字典结构: 索引 0=blank, 1=空格, 2~18384=18383个字符 (含中/英/数字/标点/emoji)

3. 技术架构

输入图片 (BGR)
  ├─→ [检测: resize 960 → ONNX → DBPostProcess] → 30个文本框
  └─→ [识别: 逐框裁剪 → resize 48×W → ONNX → CTCLabelDecode] → 文字
输出: [(文字, 置信度, 坐标), ...] → PIL 中文渲染 → 可视化

5. 快速开始

git clone https://github.com/tydfgt/ocronnx.git
cd ocronnx
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python ocr_opencv_dnn.py -i test.jpg -o result.jpg

环境要求

设备Jetson Orin Nano / AGX Orin
JetPack6.x / L4T R36.x
OpenCV4.5+ CUDA 编译 (非 pip 版)
Python3.10+
⚠️ 不能 pip install opencv-python！会覆盖系统 CUDA 版。脚本已自动处理。

8. FAQ

Q: 能批处理吗？

不能。模型内部 Concat 操作要求 batch 维度一致，变长输入不兼容。

Q: TensorRT 呢？

Jetson 8GB 上 TRT 编译极慢（2-10 分钟），opt_level=2 输出错误，不实用。

Q: 中文显示不了？

已用 PIL + NotoSerifCJK 修复。cv2.putText 不支持中文。

Q: NumPy 报错？

pip install 'numpy<2' — 系统 OpenCV 用 NumPy 1.x 编译。

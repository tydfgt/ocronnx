🔧 技术栈

模型PP-OCRv5 Mobile (DBNet 检测 + SVTR 识别)
推理引擎OpenCV 4.10.0 DNN + CUDA 12.6
模型格式ONNX (PIR → paddle.onnx.export → onnxsim)
后处理PaddleOCR 原生 DBPostProcess + CTCLabelDecode
平台NVIDIA Jetson Orin Nano (JetPack 6 / L4T R36.5)

🚀 快速开始

# 克隆仓库
git clone https://github.com/tydfgt/ocronnx.git
cd ocronnx

# 创建环境
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 推理
python ocr_opencv_dnn.py -i test.jpg -o result.jpg

# 性能测试
python ocr_opencv_dnn.py -i test.jpg --benchmark

📁 项目结构

ocronnx/
├── ocr_opencv_dnn.py          # 主推理脚本
├── export_to_onnx.py          # PIR → ONNX 导出

├── README.md                  # 项目说明

📖 命令行参数

python ocr_opencv_dnn.py --help

  -i, --image    输入图片路径 (必填)
  -o, --output   输出图片路径 (默认: output.jpg)
  --det_size     检测 resize 边长 (默认: 960, 640 更快)
  --benchmark    性能测试模式
  --cpu          使用 CPU 推理 (默认 CUDA)

⚙️ 模型导出 (可选)

python export_to_onnx.py

❓ FAQ

Q: 为什么不用 PaddlePaddle GPU？

PaddlePaddle 官方未发布 JetPack 6 (L4T 36.x) 的 GPU 预编译包，仅支持到 JetPack 5.x。

Q: 能批处理吗？

当前模型架构限制 batch > 1，OpenCV DNN 会报 Concat 维度错误。

Q: TensorRT 呢？

Jetson Orin Nano 8GB 上 TRT 引擎编译极慢（检测 ~2min, 识别 ~10min），不实用。

Q: 中文显示不了？

已用 PIL + NotoSerifCJK 字体修复。系统需安装 fonts-noto-cjk。

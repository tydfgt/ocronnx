4. DBNet 文字检测原理

DBNet = Differentiable Binarization Network（可微分二值化网络）。本质是一种基于图像分割的文字检测方法。

4.1 什么是图像分割？

图像分割就是把图片中每个像素分类。比如：

输入图片 (H×W×3)
   ↓ CNN
输出 (H×W×1) — 每个像素一个值:
   0.0 = 背景
   1.0 = 文字
   中间值 = 可能是文字

类比电器工程：就像热成像仪——每个像素一个温度值，高温区就是"文字区域"。

4.2 DBNet 处理流程 (逐步详解)

Step 1: 预处理

原图 (BGR, H×W×3)
  → 长边缩放到 960px, 保持宽高比
  → 短边 padding 到 32 的倍数 (DBNet 下采样 1/32)
  → 归一化: mean=[0.485,0.456,0.406] std=[0.229,0.224,0.225]
  → HWC → CHW → BCHW

Step 2: 特征提取 (Backbone)

MobileNetV3 卷积神经网络逐层提取特征。低层检测边缘，高层检测文字区域。

类比：就像用不同分辨率的放大镜看 PCB——低倍看全局布局，高倍看焊点细节。

Step 3: 多尺度融合 (FPN Neck)

FPN (Feature Pyramid Network) 把不同层次的特征融合在一起，既保留细节（小文字）又保留全局信息（大文字）。

Step 4: 输出概率图

网络输出一张 (1×H×W) 的概率图，每个像素值 0~1 表示该像素是文字的概率。

  • 0.98 → 99%确定是文字
  • 0.05 → 几乎确定是背景
  • 0.52 → 不确定

Step 5: 二值化

概率图 > 0.3 → 1 (文字)
概率图 ≤ 0.3 → 0 (背景)

阈值 0.3 是经验值（来自 PP-OCRv5 配置）。

4.3 从二值图到文本框

二值图 (0/1)
  ↓ cv2.findContours()
  ↓ 找到所有白色区域的轮廓
轮廓列表 (每个轮廓是一组坐标点)
  ↓ cv2.approxPolyDP()
  ↓ 用多边形近似轮廓
  ↓ 过滤掉太小的 (< 3px) 或置信度低的
  ↓ Vatti 裁剪算法 (unclip)
  ↓ 扩展文本框 (ratio=2.0)
最终文本框 (四边形坐标)

Vatti Unclip 算法

为什么需要"扩展"？卷积网络倾向输出"收缩"的文字区域（只覆盖文字核心部分）。unclip 把每个框向外扩展一定距离：

d = area × ratio / perimeter
  • ratio=1.5: 轻度扩展
  • ratio=2.0: 中等扩展 (本项目用)
  • ratio=3.0: 激进扩展 (可能合并相邻框)
本项目用 unclip_ratio=2.0 来补偿 ONNX 模型输出的轻微碎片化。

4.4 关键参数

参数默认值作用
thresh0.3二值化阈值。越大越保守（少框），越小越激进（多框）
box_thresh0.6文本框最低置信度。低于此值的框被丢弃
unclip_ratio2.0文本框扩展比例。越大框越大
resize_long960长边缩放尺寸。越大越准但越慢
max_candidates1000最多检测框数量

4.5 为什么输入必须是 32 的倍数？

DBNet 的 MobileNetV3 backbone 有 5 次 2× 下采样（共 1/32）。如果输入尺寸不能被 32 整除，某些层的特征图尺寸不一致，Add 操作会报错：

ERROR: input[0]=[1,96,71,120] input[1]=[1,96,72,120]
→ 71 ≠ 72, Add 失败!
本项目在预处理中自动 padding 到 32 的倍数：pad_h = (32 - h % 32) % 32

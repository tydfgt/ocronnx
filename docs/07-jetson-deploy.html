7. Jetson 部署方案对比

在 Jetson Orin Nano 8GB 上部署 PP-OCRv5，可选的推理方案对比。

7.1 四种方案对比

方案速度精度部署难度JetPack 6推荐
C++ Paddle 推理5.5s/图✅ 基准高❌ 不支持❌
Python PaddleOCR CPU~15s/图✅ 基准低⚠️ CPU Only❌
ONNX + OpenCV DNN CUDA~1.0s/图✅ 一致中✅✅
TensorRT未知(构建慢)❌ 输出全0极高⚠️ 部分❌
🏆 最终方案: ONNX + OpenCV DNN CUDA — 6× 快于 C++ 方案，精度与基准一致，且在 JetPack 6 上可用。

7.2 环境要求

组件版本说明
Jetson Orin Nano8GB SuperAmpere GPU, 1024 CUDA cores
L4TR36.5.0Linux for Tegra
JetPack6.2含 CUDA 12.6, cuDNN 9.3
OpenCV4.10.0 (系统)含 CUDA DNN 模块
Python3.10系统自带
NumPy1.26.4需 <2.0 兼容系统 OpenCV
PaddleOCR3.6.0仅用于后处理和字典

7.3 安装步骤

# 1. 创建虚拟环境
python3 -m venv venv_paddleocr
source venv_paddleocr/bin/activate

# 2. 安装 PaddlePaddle CPU (仅用于模型导出和后处理)
pip install paddlepaddle==3.2.2

# 3. 安装 PaddleOCR
pip install paddleocr==3.6.0

# 4. 降级 NumPy (兼容系统 OpenCV)
pip install 'numpy<2'

# 5. 卸载 pip 版 opencv-python (会覆盖系统 CUDA 版)
pip uninstall opencv-python opencv-python-headless -y

# 6. 安装 ONNX 工具
pip install onnx onnxsim paddle2onnx==2.1.0

# 7. 导入系统 OpenCV
python -c "import sys; sys.path.insert(0,'/usr/local/lib/python3.10/dist-packages'); import cv2; print(cv2.__version__)"
# 应输出: 4.10.0

7.4 性能分析

阶段耗时占比设备
检测推理 (DBNet)130ms13%GPU
检测后处理20ms2%CPU
识别推理 (SVTR)790ms79%GPU
识别后处理10ms1%CPU
绘图 & I/O50ms5%CPU
总计~1000ms100%

瓶颈分析：97% 时间在 GPU 推理上。识别远比检测慢（每个检测框要跑一次识别模型）。

⚠️ 当前不支持 batch 推理。识别模型的 Concat 算子要求 batch 内尺寸一致，而不同文字区域宽度不同，无法 batch。若后续模型改进支持 batch，可提速 5-8×。

7.5 为什么 TensorRT 失败？

TensorRT 构建 ONNX→TRT 引擎时发现：

  • optimization_level=2: 生成结果正确，但推理输出全为 0。疑似 TRT 在 Jetson 上优化过度导致计算错误。
  • optimization_level=3: 构建引擎需 3+ 分钟（检测）+ 更长时间（识别），无法承受。
  • ONNX 模型中的动态 shape 和 Reshape 算子与 TRT 兼容性不佳。
结论：对 Jetson Orin Nano，OpenCV DNN 是 ONNX 的最佳推理引擎。TRT 更适合 x86 + 高端 RTX GPU。

7.6 运行时命令

# 单张图片推理
python ocr_opencv_dnn.py --image input.jpg --output result.jpg

# 自定义检测尺寸 (越大越准但越慢)
python ocr_opencv_dnn.py --image input.jpg --det_size 640

# 性能基准测试
python ocr_opencv_dnn.py --image input.jpg --benchmark

# CPU 模式 (无 GPU 或调试)
python ocr_opencv_dnn.py --image input.jpg --cpu
